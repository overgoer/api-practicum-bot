# API Practicum Bot - Telegram бот для воронки продаж

## 🎯 Назначение проекта
Telegram-бот для воронки продаж онлайн-практикума по тестированию API. Бот проводит пользователей через мини-практикум с задачами по поиску багов в API, подготавливая к покупке полного курса.

## 🏗 Архитектура
- **Фреймворк**: aiogram 3.x (асинхронный)
- **Язык**: Python 3.11+
- **База данных**: SQLite3 (файл practicum.db)
- **Состояния**: FSM (Finite State Machine) для управления диалогом
- **Паттерн**: Обработчики сообщений и колбэков с использованием декораторов

## 🚀 Инструкции по запуску

### 1. Установка зависимостей
```bash
pip install aiogram==3.4.1
```

### 2. Настройка токена бота
В файле `main.py` замените значение переменной `TOKEN` на актуальный токен от @BotFather:
```python
TOKEN = "ВАШ_ТОКЕН_БОТА"
```

### 3. Инициализация базы данных
База данных автоматически создается при первом запуске бота.

### 4. Запуск бота
```bash
python main.py
```

## 📊 База данных
Таблица `users` отслеживает прогресс пользователей:
- `tg_id`: ID пользователя в Telegram (первичный ключ)
- `username`: Имя пользователя
- `step`: Текущий шаг в воронке (start, task1_view, task2_view, task2_done)
- `last_act`: Время последней активности (TIMESTAMP)

Функция `upsert_user()` обновляет или создает запись пользователя при каждом взаимодействии.

## 🔄 Диалоговая модель (FSM)
Бот использует FSM с тремя состояниями:
1. `Practicum.welcome` - Приветствие и начало
2. `Practicum.task1` - Задача 1 (проверка статус-кода)
3. `Practicum.task2` - Задача 2 (валидация данных)

## 🐛 Задачи мини-практикума

### Задача 1: Проверка ответа API
- **Запрос**: GET /api/v1/profile с валидным токеном
- **Ответ**: 200 OK с телом {"user_id": 456}
- **Вопрос**: Где баг?
- **Правильный ответ**: Бага нет (статус 200 корректен для валидного токена)

### Задача 2: Валидация данных
- **Запрос**: POST /api/v1/orders с телом {"product_id": 123, "quantity": -1}
- **Ответ**: 200 OK
- **Вопрос**: Что не так?
- **Правильный ответ**: Ошибка 400 Bad Request (отрицательное количество товара)

## 📈 Воронка продаж
Мини-практикум является первой ступенью воронки:
1. **Бесплатные задачи в боте** → Демонстрация ценности тестирования API
2. **Доказательство ценности** → Пользователь видит свои ошибки и понимает необходимость обучения
3. **Оффер полного практикума** → Переход на платный курс через ссылку в Telegram-канал

## 📝 Примечания для ИИ-агентов

### Архитектурные особенности
1. Бот использует асинхронную архитектуру aiogram 3.x
2. Все состояния хранятся в FSMContext и дублируются в БД
3. Пользовательские данные сохраняются при каждом взаимодействии
4. Используются inline-кнопки для навигации

### Контекст воронки продаж
1. Бот является точкой входа в продуктовую воронку
2. Задачи подобраны для демонстрации типичных ошибок в API
3. После завершения мини-практикума пользователь перенаправляется в Telegram-канал
4. Ссылка на канал в коде требует актуализации (сейчас: https://t.me/your_channel)

### Рекомендации по развитию

2. Реализовать систему достижений или бейджей
3. Добавить аналитику по прохождению задач
4. Интегрировать с CRM для отслеживания конверсий
