**2026-04-21 00:05:00** - Создание сервисного файла
- Создан файл: `/etc/systemd/system/api-practicum-bot.service`
- Содержимое сервиса:
  ```ini
  [Unit]
  Description=API Practicum Telegram Bot
  After=network.target
  
  [Service]
  Type=simple
  User=claude_agent
  Group=claude_agent
  WorkingDirectory=/home/claude_agent/projects/api_practicum_bot
  ExecStart=/usr/bin/python3 /home/claude_agent/projects/api_practicum_bot/main.py
  Restart=always
  RestartSec=10
  StandardOutput=journal
  StandardError=journal
  Environment=PYTHONUNBUFFERED=1
  
  [Install]
  WantedBy=multi-user.target
  ```

## 🏆 Рекомендации по лучшей конфигурации

### Вариант 1: Systemd сервис (рекомендуется)
**Преимущества:**
- Стандартный способ для Linux-серверов
- Автоматический запуск при загрузке системы
- Автоперезапуск при падениях (Restart=always)
- Интеграция с journald для централизованного логирования
- Управление через systemctl (start/stop/restart/status)
- Безопасность через sandboxing опции (NoNewPrivileges, ProtectSystem и т.д.)

**Пример конфигурации:**
```ini
[Unit]
Description=API Practicum Telegram Bot
After=network.target
Wants=network.target

[Service]
Type=simple
User=claude_agent
WorkingDirectory=/home/claude_agent/projects/api_practicum_bot
ExecStart=/usr/bin/python3 main.py
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal

# Защита
NoNewPrivileges=true
ProtectSystem=strict
ProtectHome=true
PrivateTmp=true

[Install]
WantedBy=multi-user.target
```

### Вариант 2: PM2 (альтернатива)
**Преимущества:**
- Простота настройки для Node.js/Python приложений
- Встроенный мониторинг и dashboard
- Кластерный режим (если потребуется масштабирование)
- Гибкая конфигурация через ecosystem file

**Недостатки:**
- Дополнительная зависимость (Node.js)
- Менее стандартизирован, чем systemd

### Вариант 3: Docker контейнер
**Преимущества:**
- Полная изоляция окружения
- Простота развёртывания и масштабирования
- Версионность образов
- Оркестрация через docker-compose

**Недостатки:**
- Overhead для простого бота
- Требует настройки Docker на сервере
- Сложнее отладка

### Рекомендация:
**Использовать systemd сервис** как наиболее подходящий вариант для текущей инфраструктуры:
1. Уже есть опыт использования systemd для других ботов на сервере
